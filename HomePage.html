Introduction

A VPC is a virtually isolated network in the cloud, which resembles a traditional network in a corporate datacenter. Users have complete control over their virtual networking environment, including a selection of own IP address range, creation of subnets, and configuration of route tables and network gateways. For choosing a VPC and creating its subnets, you need to select the range of private IP addresses of the VPC and its subnets called Classless Inter-domain Routing (CIDR). This IP range will be further allocated to all the resources inside this VPC. Consider a CIDR IPv4 range 10.10.0.0/16 Here /16(called as the netmask) will allocate 65536 IPv4 addresses in VPC using the calculation given below: As IPv4 represents 32 bit addressing, 32-16(n)= 16 ü°™ 2^16 (2 raised to power 16) = 65536. The range can vary from 10.10.0.0 to 10.10.256.256. The VPC block size is in between /28 and /16 netmask

You can easily customize the network configuration for your Amazon VPC. For example, you can create a public-facing subnet for your web servers that has access to the Internet, and place your backend systems such as databases or application servers in a private-facing subnet with no Internet access. You can leverage multiple layers of security, including security groups and network access control lists, to help control access to Amazon EC2 instances in each subnet.
